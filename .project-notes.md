# TCG Ledger - Project Notes

Last updated: 2026-04-20

## Overview
Self-hosted TCG collection tracker + trade binder for Magic and Pokémon.
Runs on Raspberry Pi 5 (192.168.1.20, Tailnet 100.65.248.44).
Public via Tailscale Funnel at https://raspberrypi.tail884296.ts.net

## Deployment
- Build on Fedora, rsync to Pi, Docker Compose (Postgres 16 + Node 22)
- Build requires dummy env vars: `DATABASE_URL="postgres://fake:fake@localhost:5432/fake" AUTH_SECRET="build-time-placeholder-min-32-chars-long"`
- Prisma binary target: `linux-arm64-openssl-3.0.x` (bookworm-slim)
- App container installs openssl on startup (command in docker-compose.yml)
- Prod .env on Pi at /opt/tcg-ledger/.env (0600, owned by melkor)

## Database
- Prod: Pi Postgres, not port-exposed. Access via `docker compose exec -T postgres psql -U tcg -d tcg_ledger`
- Dev: localhost:5433 via podman-compose (named volume, not bind mount - rootless Podman fix)
- SSH tunnel for remote Prisma access: `ssh -f -N -L 15432:172.20.0.2:5432 melkor@192.168.1.20`
  (172.20.0.2 is the postgres container IP on Docker bridge network)
- Catalog: 96K Magic (Scryfall) + 20K Pokémon cards seeded via scripts/seed-catalog.ts

## Auth
- NextAuth v5 JWT, credentials provider only
- Edge middleware needs `secret` passed explicitly (AUTH_TRUST_HOST=true in compose)
- Self-registration creates USER role with addCardsAccess=true, inventoryUpdatePrices=false
- Admin account on prod: abhik107@gmail.com (password was set to admin123, should be changed)

## Architecture decisions
- Card uniqueness: (name, set, collectorNumber, finish) - works for both games
- Holding uniqueness includes condition - separate rows for NM vs LP
- marketPrice on Card (shared), acquiredPrice on Holding (personal)
- listedQuantity on Holding for partial trade listings
- Ledger: append-only quantityLog + priceLog, UNION ALL query for combined view
- Raw SQL column names: quantityLog uses cardname (lowercase via @map), cardSet/userId/actorId are camelCase
